## Overview

The parameters system (parametros) provides a flexible classification mechanism for products and suppliers. Create custom status parameters with visual color coding to track inventory states, product conditions, or any other categorical data you need.

## Key Features

- Create unlimited custom parameters
- Color-coded visual identification
- Automatic code generation
- Real-time product count tracking
- Integration with products and suppliers
- Deletion protection for parameters in use
- Predefined color palette with 10 options

## Data Structure

| Field | Type | Required | Max Length | Description |
|-------|------|----------|------------|--------------|
| `id` | Integer | Auto | - | Unique identifier |
| `codigo` | String | Auto/Manual | - | Parameter code (e.g., "001", "002") |
| `nombre` | String | Yes | 50 chars | Parameter name/label |
| `descripcion` | String | No | - | Optional description |
| `color` | String | Yes | 7 chars | Hex color code (#RRGGBB format) |
| `fecha_creacion` | DateTime | Auto | - | Creation timestamp |
| `fecha_actualizacion` | DateTime | Auto | - | Last update timestamp |

### Creating a Parameter

<Steps>
  <Step title="Open create dialog">

  </Step>
  
  <Step title="Enter parameter details">
    Fill in the required fields:

    - **Descripción**: Optional details about when to use this parameter
  </Step>
  
  <Step title="Select a color">
    Click one of the 10 color swatches in the palette:
    - Blue (#3B82F6) - Default
    - Green (#10B981)
    - Amber (#F59E0B)
    - Purple (#8B5CF6)
    - Red (#EF4444)
    - Cyan (#06B6D4)
    - Lime (#84CC16)
    - Orange (#F97316)
    - Pink (#EC4899)
    - Indigo (#6366F1)
  </Step>
  
  <Step title="Save parameter">
    Click **Guardar**. The system:
    - Auto-generates the next sequential code (e.g., "001", "002", "003")
    - Validates the color format (#RRGGBB)
    - Validates name uniqueness
    - Creates the parameter
  </Step>
</Steps>

<Info>
  The `codigo` field is automatically generated as a 3-digit sequential number (001, 002, etc.). You don't need to provide it.
</Info>

### Editing a Parameter

<Steps>
  <Step title="Locate parameter">
    Use the search bar to filter by code, name, or description. Or browse the table rows.
  </Step>
  
  <Step title="Click edit button">
    Click **Editar** in the actions column of the parameter you want to modify.
  </Step>
  
  <Step title="Update fields">
    Modify the name, description, or color. The code cannot be changed after creation.
  </Step>
  
  <Step title="Save changes">
    Click **Guardar**. The same validation rules apply.
  </Step>
</Steps>

<Note>
  You cannot change the `codigo` field after creation. Only `nombre`, `descripcion`, and `color` can be updated.
</Note>

### Deleting a Parameter

<Warning>
  You cannot delete a parameter that is currently assigned to products. The system will block deletion and show the number of associated products.
</Warning>

<Steps>
  <Step title="Check product count">
    Look at the "Productos" column in the table. If the count is greater than 0, you must reassign those products first.
  </Step>
  
  <Step title="Click delete button">
    Click **Eliminar** in the actions column.
  </Step>
  
  <Step title="Confirm deletion">
    A modal appears showing the parameter name. Click **Confirmar** to proceed.
  </Step>
  
  <Step title="Validation check">
    The backend verifies no products are using this parameter:
    - If products exist, deletion is blocked with error message
    - If no products exist, parameter is permanently deleted
  </Step>
</Steps>

## Product Integration

Parameters are tightly integrated with the products table:

### Product Count Tracking

**Source:** `parametros.php:16-27`

The GET endpoint performs a LEFT JOIN to count products:

```php
SELECT par.id, par.codigo, par.nombre, par.descripcion, par.color,
       COUNT(prod.id) AS productos_count
FROM parametros par
LEFT JOIN productos prod ON prod.parametro_id = par.id
GROUP BY par.id, par.codigo, par.nombre, par.descripcion, par.color
ORDER BY par.id
```

This provides real-time product counts without additional queries.

### Deletion Protection

**Source:** `parametros.php:98-101`

```php
$cnt = 0;
$rc = $conn->query("SELECT COUNT(*) AS c FROM productos WHERE parametro_id=$id");
if ($rc) { 
  $cnt = intval($rc->fetch_assoc()['c']); 
}
if ($cnt > 0) { 
  echo json_encode([
    'error' => 'No se puede eliminar: tiene productos asociados'
  ]);
  exit;
}
```

This prevents orphaned products by blocking parameter deletion.

## Validation Rules

<AccordionGroup>
  <Accordion title="Nombre Validation">
    **Required Field**
    
    - Cannot be empty
    - Maximum 50 characters
    - Must be unique across all parameters
    - Trimmed of whitespace
    
    **Frontend:** `parametros.js:199`
    ```javascript
    if (!body.nombre) {
      mostrarAlerta('El estado es obligatorio', 'error');
      return;
    }
    ```
    
    **Backend:** `parametros.php:38-39`
    ```php
    if ($nombre === '') {
      echo json_encode(['error'=>'El nombre es obligatorio']);
    }
    if (strlen($nombre) > 50) {
      echo json_encode(['error'=>'Longitud de nombre inválida']);
    }
    ```
  </Accordion>
  
  <Accordion title="Color Validation">
    **Hex Format Required**
    
    - Must match pattern: `^#[0-9A-Fa-f]{6}$`
    - Exactly 7 characters: # followed by 6 hex digits
    - Default: #3B82F6 if not provided
    
    **Backend:** `parametros.php:41`
    ```php
    if (!preg_match('/^#[0-9A-Fa-f]{6}$/', $color)) {
      echo json_encode(['error'=>'Color inválido. Use formato #RRGGBB']);
    }
    ```
    
    Valid examples:
    - ✅ `#3B82F6`
    - ✅ `#10b981` (lowercase accepted)
    - ❌ `3B82F6` (missing #)
    - ❌ `#3B8` (too short)
    - ❌ `#GGGGGG` (invalid hex)
  </Accordion>
  
  <Accordion title="Codigo Auto-Generation">
    **Automatic Sequential Codes**
    
    If codigo is empty, the system generates the next sequential 3-digit code:
    
    **Backend:** `parametros.php:43-47`
    ```php
    if ($codigo === '') {
      $resNext = $conn->query(
        "SELECT LPAD(COALESCE(MAX(CAST(codigo AS UNSIGNED)),0)+1,3,'0') 
         AS next_codigo FROM parametros"
      );
      $codigo = $rowNext['next_codigo'] ?? '001';
    }
    ```
    
    This ensures:
    - First parameter gets "001"
    - Next gets "002", then "003", etc.
    - Leading zeros are preserved
    - Gaps from deletions are not filled
  </Accordion>
  
  <Accordion title="Uniqueness Validation">
    **Unique Codigo and Nombre**
    
    Both codigo and nombre must be unique:
    
    **Backend:** `parametros.php:53-54`
    ```php
    $chk = $conn->query(
      "SELECT 1 FROM parametros 
       WHERE codigo='$codigo' OR nombre='$nombre' LIMIT 1"
    );
    if ($chk && $chk->num_rows > 0) {
      echo json_encode(['error'=>'Código o nombre ya existe']);
    }
    ```
    
    For updates (excludes current record):
    ```php
    WHERE nombre='$nombre' AND id!=$id
    ```
  </Accordion>
  
  <Accordion title="Deletion Validation">
    **Product Association Check**
    
    Cannot delete if products reference this parameter:
    
    **Backend:** `parametros.php:98-101`
    ```php
    $cnt = $conn->query(
      "SELECT COUNT(*) AS c FROM productos WHERE parametro_id=$id"
    )->fetch_assoc()['c'];
    
    if ($cnt > 0) {
      echo json_encode([
        'error' => 'No se puede eliminar: tiene productos asociados'
      ]);
    }
    ```
  </Accordion>
</AccordionGroup>

### Backend (parametros.php)

**Database Table:** `parametros`

**Key Operations:**

- **Line 16-27**: GET with LEFT JOIN and GROUP BY for product counts
- **Line 30-60**: POST - Auto-generates codigo, validates color format
- **Line 43-47**: Sequential code generation logic
- **Line 53-54**: Uniqueness check for codigo and nombre
- **Line 63-88**: PUT - Updates all fields except codigo
- **Line 91-107**: DELETE - Validates no products before deletion

**Code Generation Algorithm:**

```php
// Find maximum existing code, cast to integer, add 1, pad to 3 digits
SELECT LPAD(
  COALESCE(MAX(CAST(codigo AS UNSIGNED)), 0) + 1,
  3,
  '0'
) AS next_codigo 
FROM parametros
```

This ensures:
- "001" for first parameter
- "002", "003", etc. for subsequent ones
- Handles deletions gracefully (doesn't reuse codes)

## Common Error Messages

| Error Message | Cause | Solution |
|---------------|-------|----------|
| "El estado es obligatorio" | Empty nombre field | Enter a parameter name |
| "Longitud de nombre inválida" | Name exceeds 50 characters | Shorten the name |
| "Color inválido. Use formato #RRGGBB" | Invalid hex color | Use format #3B82F6 |
| "Código o nombre ya existe" | Duplicate codigo or nombre | Choose unique values |
| "No se puede eliminar: tiene productos asociados" | Products using this parameter | Reassign products first |
| "No encontrado" | Invalid parameter ID | Refresh page and retry |
| "Datos inválidos" | Malformed JSON request | Check request format |
| "ID y nombre son obligatorios" | Missing required fields on update | Provide id and nombre |
